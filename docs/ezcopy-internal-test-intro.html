把素材安安静静地送到电脑上

拍完素材之后，最烦的往往不是剪辑，而是“怎么把这一堆视频和照片弄到电脑里”。EZCopy 做的事情很简单：让 iPhone 和电脑在同一个 Wi-Fi 下直接传文件。不开云盘，不绕服务器，不走蜂窝流量，素材不出去旅行。

内部测试版本重点验证局域网连接、ZIP 下载、访问码安全、元数据报告和大文件体验。请尽量在真实拍摄后的素材场景里测试它。

它适合什么场景

EZCopy 不是大而全的同步工具，它更像拍摄现场的“素材搬运小推车”：目标明确，路线短，最好别绕路。

Creator Workflow
Scene 01

拍摄结束快速收工

手机里刚拍完一批短视频素材，电脑就在旁边。选择文件，复制链接，电脑下载 ZIP，今天的素材先落袋为安。

Scene 02

没有外网也能传

外拍现场、临时路由器、工作室内网，只要 iPhone 和电脑在同一局域网，就不必追问“这里网速怎么样”。

Scene 03

电脑端不想装软件

Chrome 或 Edge 就是接收端。少装一个东西，就少一个“这东西到底干嘛的”的解释环节。

Scene 04

素材不想经过第三方

不先丢网盘，不先发聊天工具，不先让素材出去逛一圈。文件在你的设备之间传。

重点功能

这次内部测试请优先感受这些功能是否顺手。尤其是 Address、Access Code、Copy Link 这一区域，它决定了第一次上手有没有挫败感。

Feature Focus
只走局域网

核心传输发生在同一 Wi-Fi / 局域网内，不需要云端服务器。

一键复制链接

PC 扫码有时像在给显示器做眼保健操。EZCopy 直接复制完整链接，更符合日常使用。

短访问码

手动输入只需要基础地址加 6 位 Access Code，安全感保留，痛苦值下降。

自动打包 ZIP

照片、视频、Manifest、报告一次打包，电脑端按浏览器下载流程保存。

可选 MD5

需要完整性校验就开启；赶时间传大视频时可以关闭，少跑一段认真但有点慢的流程。

缓存可清理

临时 ZIP 会在传输后清理，也可以在 App 中手动清理缓存。

推荐测试流程

请尽量用真实素材测试，比如一组手机拍摄的照片和几段视频。空素材测试会很快，但它不太会暴露真正的问题。

How To Test

连接同一个 Wi-Fi

iPhone 和电脑需要处于同一局域网。这个 Wi-Fi 是否能上互联网不是核心条件。

选择照片和视频

在 EZCopy 中从相册选择要传输的手机拍摄素材。

启动 Receiver

点击 Start Receiver，App 会显示 Address、Access Code 和过期时间。

打开电脑浏览器

用 Chrome 或 Edge 打开复制的完整链接，或者手动输入 Address 后再输入 Access Code。

下载素材包

点击 Download EZCopy Package，等待 ZIP 下载完成。

解压并检查

确认媒体文件、Manifest、报告和可选 MD5 文件是否完整，文件名和基础元数据是否符合预期。

安全与边界

这里的目标是实用安全：不把素材送上网，同时避免同网段里随便一个人都能进来。

  • 传输链路在局域网内完成，不主动上传素材到互联网。
  • 媒体文件不使用蜂窝数据传输。
  • 每次接收会话都有新的 6 位 Access Code。
  • 会话默认 10 分钟过期，过期后需要重新启动 Receiver。
  • Access Code 错误或过期会被拒绝访问。
  • MD5 用于完整性检查，不是加密功能。

请注意

“不依赖互联网”这句话基本成立，但有一个重要前提：你要传的原片已经在 iPhone 本机。

Correction
  • 如果照片或视频原片只在 iCloud 里，EZCopy 当前不会自动联网下载。
  • 需要用户先在相册中把 iCloud 原片下载到本机，再进行传输。
  • 当前仅支持 Chrome 和 Edge，不支持 Safari、Firefox。
  • 当前基础档采用 ZIP 下载方式，不是直接写入电脑文件夹。
  • 大视频打包前，iPhone 需要有足够临时空间。
  • 公共 Wi-Fi 环境仍建议谨慎使用，不要把链接发到无关群聊里。

内部测试重点

如果只想反馈最有价值的问题，请优先围绕下面这些点。比起“感觉还行”，我们更想知道它在哪一步卡住、慢了、看不懂或者让你皱眉。

Feedback Checklist
连接成功率

同一 Wi-Fi 下，电脑能否顺利打开 Address？不同路由器环境表现如何？

输入体验

Access Code 是否好认、好抄？Copy Link 是否明显、可靠？

大文件表现

长视频打包和下载是否稳定？手机发热、耗电、空间占用是否可接受？

文件完整性

ZIP 解压后文件是否完整？视频能否正常播放？文件名是否保留？

报告准确性

Manifest 和 HTML 报告里的文件名、时长、尺寸、拍摄时间是否正确？

异常提示

输错 Access Code、会话过期、断网、iCloud 原片未下载时，提示是否看得懂？
